> アプリケーションと CSV ファイルに関する基本的な知識のみを前提とします。

> **範囲:** RABET 1.3.2 のすべての機能を網羅しています。英語版は別ファイル
> [`USER_GUIDE.md`](USER_GUIDE.md) として用意されています。

## 1. はじめに

### 1.1 RABET をダウンロードする

RABET は **Zenodo を公式の配布元** としており、リリースごとに引用可能な永続
DOI が付与されます。

> **Zenodo（公式）**: https://doi.org/10.5281/zenodo.15313025
> *(コンセプト DOI — 常に最新版を指します)*

Zenodo のレコードを開き、お使いのプラットフォームに対応するアーカイブを
ダウンロードしてください。

| プラットフォーム | アセット |
| --- | --- |
| Windows | `RABET-Windows-1.3.2.zip` |
| macOS (Apple Silicon) | `RABET-macOS-arm64-1.3.2.zip` |
| macOS (Intel) | `RABET-macOS-x86_64-1.3.2.zip` |
| Linux | `RABET-Linux-x86_64-1.3.2.tar.gz` |

同じバイナリのミラーが
[GitHub Releases](https://github.com/mi2e-K/RABET/releases) でも配布されています。
どちらをダウンロードしても中身は同じです。

### 1.2 RABET を起動する

#### Windows

1. アーカイブ（`RABET-Windows-1.3.2.zip`）を任意の場所に展開します。
   デスクトップでも、専用の `Tools\` フォルダでも構いません。
2. 展開した `RABET` フォルダを開き、**`RABET.exe`** をダブルクリックします。
   ショートカットを使いたい場合は同梱の **`Launch RABET.bat`** でも構いません。
3. 初回起動時に Windows SmartScreen が「PC が保護されました」と表示する
   ことがあります。これは実行ファイルがコード署名されていないためです。
   *詳細情報* → *実行* を選んで進めてください。

#### macOS

1. ご利用の CPU アーキテクチャに合うアーカイブを展開します
   （Apple Silicon = M1/M2/M3/M4、Intel = 2020 年以前の Mac）。
2. 展開した **`RABET.app`** を `Applications` フォルダへ移動します。
3. バンドルが署名されていないため、初回起動だけ右クリック → **開く** で
   起動してください。2 回目以降は通常通り起動できます。

#### Linux

1. tarball を展開します。
   ```bash
   tar -xzf RABET-Linux-x86_64-1.3.2.tar.gz
   ```
2. 展開したディレクトリに入り、次のコマンドで起動します。
   ```bash
   ./run_rabet.sh
   ```
3. アプリケーションメニューに RABET を登録する場合は、次を実行します。
   ```bash
   ./install_desktop_entry.sh
   ```

### 1.3 初回起動時に作成されるフォルダ

RABET は初回起動時に、OS のユーザーアプリケーションデータディレクトリ配下に
3 つのフォルダを作成します。

- **`configs/`** — アクションマップ、カスタムメトリクス、配色の JSON
- **`logs/`** — 実行ログ（`rabet_<日付>.log`）。トラブルシューティングに使用
- **`projects/`** — 新規プロジェクトのデフォルト保存先

具体的なパスは OS により異なります。

| OS | 場所 |
| --- | --- |
| Windows | `%APPDATA%\RABET\` |
| macOS | `~/Library/Application Support/RABET/` |
| Linux | `~/.config/RABET/` |

RABET は各ファイルダイアログで最後に使ったディレクトリも記憶し、次回以降
そこを既定として開きます。

### 1.4 メインウィンドウの構成

アプリケーションウィンドウは次のように構成されています。

```
┌──────────────────────────────────────────────────────────────────┐
│  File   Edit   View   Log   Help                                 │  ← メニューバー
├──────────────────────────────────────────────────────────────────┤
│  [Annotation] [Analysis] [Visualization] [Reliability] [Project] │  ← モードタブ
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│                  （アクティブなモードのメイン画面）                 │
│                                                                  │
├──────────────────────────────────────────────────────────────────┤
│  ステータスバー：簡易メッセージ、現在ファイル、録画状態               │
└──────────────────────────────────────────────────────────────────┘
```

**モードタブ** は RABET の 5 つの主要ワークフローに対応します。**View**
メニューから、またはタブを直接クリックして、いつでも切り替えられます。
アクティブなモードは強調表示されます。

## 2. 動画にアノテーションを付ける

**Annotation** ビューは RABET の中核となる作業画面です。ビデオプレイヤー、
タイムライン、録画操作パネル、アクションマップパネルがひとつの画面に
まとまっています。

```
┌───────────────────────────────────────────────────────────────────┐
│                                                  │ アクションマップ │
│            ビデオプレイヤー（フレーム精度）         │ ┌────┬───────┐ │
│                                                  │ │キー│  行動  │ │
│                                                  │ ├────┼───────┤ │
│                                                  │ │ o  │Attack │ │
│                                                  │ │ j  │Side.  │ │
│                                                  │ │... │ ...   │ │
├──────────────────────────────────────────────────│ └────┴───────┘ │
│  録画操作 — 時間設定、Start / Pause / Stop        │ [Add][Edit]    │
├──────────────────────────────────────────────────────────────────┤
│  タイムライン — 色分けされたイベント、再生ヘッド                     │
└──────────────────────────────────────────────────────────────────┘
```

### 2.1 動画を開く

3 つの同等な方法があります。

1. **メニュー**: `File → Open Video`
2. **最近開いた動画**: `File → Open Recent Video` — 直近 10 件を一覧表示
3. **ドラッグ&ドロップ**: 動画ファイルを RABET ウィンドウの任意の場所に
   ドラッグします。自動的に Annotation モードへ切り替わります。

標準的な拡張子（`.mp4`、`.mov`、`.avi`、`.mkv`、`.webm`、`.m4v`、`.wmv`、
`.flv`、`.ts`）は拡張子で受け付けます。録画ソフトが独自の拡張子（標準
MP4 コンテナに `.video` を付けるなど）を使っている場合でも、RABET は
マジックナンバー判定と PyAV による試験的なオープンの 2 段階フォールバック
を持っているため、ファイルの中身が有効な動画なら開けます。

オープン後は最初のフレームが表示され、再生ヘッドは 0 にリセットされます。

### 2.2 アクションマップ・パネル

アクションマップは **キーボードの 1 キー** を **行動ラベル** に対応付けます。
録画中は、マップされたキーを押すと行動イベントが開始し、同じキーを離すと
終了します。

**デフォルトのマッピング**（`configs/default_action_map.json` から読み込み）:

| キー | 行動 |
| :-: | --- |
| `o` | Attack bites |
| `j` | Sideways threats |
| `p` | Tail rattles |
| `q` | Chasing |
| `a` | Social contact |
| `e` | Self-grooming |
| `t` | Locomotion |
| `r` | Rearing |

Annotation ビューの右側にあるアクションマップ・パネルには、アクティブな
マップが編集可能なテーブルとして表示されます。テーブル下のボタンで
次の操作ができます。

- **Add** — 新しい「キー → 行動」マッピングを追加します。キー欄は英数字
  1 文字を受け付けます。既存のマッピングを上書きするときは確認ダイアログ
  が出ます。
- **Edit** — 選択した行の行動ラベルを変更します。
- **Remove** — マッピングを削除します。**既に記録済みのアノテーション**
  は削除されません。

マップの保存・読み込み:

- `File → Save Action Map` で現在のマップを `.json` ファイルに保存します。
- `File → Load Action Map` で保存済みのマップを読み込みます（現在のマップ
  は置き換えられます）。
- `File → Reset Action Map to Default` で組み込みデフォルトに戻します。

### 2.3 動画のコントロール

| 操作 | ショートカット |
| --- | --- |
| 再生 / 一時停止の切替 | `Space` |
| 指定ステップ分だけ進む | `→`（右矢印） |
| 指定ステップ分だけ戻る | `←`（左矢印） |

ステップ幅と再生レートは、ビデオフレーム下のコントロール帯で変更できます。
ステップ幅のデフォルトは 1 フレームですが、一時停止中の粗い移動には
5 フレームや 10 フレームを使うのも便利です。

現在の再生位置はビデオ下に `hh:mm:ss.ms` 形式で表示されます。録画セッション
中は、同じ場所にセッション開始からの **相対時刻** も表示されます。

### 2.4 記録セッションの設定

タイムライン上部の **Recording controls** パネルで、テスト時間を `hh:mm:ss`
形式の入力欄に入れます（例: `00:05:00` で 5 分間）。

**Preserve on rewind**（巻き戻し時にイベントを保持）チェックボックスが
オンの場合、再生ヘッドが「まだ押されている最中」のイベントの開始時刻を
逆方向に超えても、そのイベントは **保持** されます。オフ（デフォルト）
の場合は **破棄** されます。「行動を数秒早くマークしすぎた」と気づいた
ときに便利です。

### 2.5 録画の状態遷移

```
        Start Recording                   Pause
   IDLE ─────────────────► WAITING ──► RECORDING ─► PAUSED
    ▲                                    ▲   │       │
    │                                    │   │ Resume│
    │                                    └───┴───────┘
    │                                    │
    │           Stop / 時間経過           │
    └────────────────────────────────────┘
```

Recording controls パネルの視覚的な手がかり:

| 状態 | ボタン表示 | 状態色 |
| --- | --- | --- |
| アイドル | **Start Recording**（緑） | （無表示） |
| 待機中 | **Cancel**（灰） | 青 |
| 録画中 | **Pause**（橙） | 赤 |
| 一時停止 | **Resume**（緑）／**Stop**（赤） | 橙 |

録画の典型的な流れ:

1. 動画を開いてテスト時間を設定します。
2. **Start Recording** をクリックします。状態が **Waiting** に変わります。
3. 動画を再生します（`Space`）。再生ヘッドが動き始めた瞬間にセッションが
   開始（状態 → **Recording**）し、その時刻が全イベントの基準時刻となります。
4. 行動キーを押してイベントを記録します。キーを離すとイベントが終了します。
5. 途中で中断したいときは **Pause** を押します。再生も停止します。**Resume**
   で再開できます。
6. テスト時間が経過するか、**Stop** を押すとセッションが終了します。

セッションが終了すると動画も自動的に一時停止されます。プロジェクト内から
録画を開始していた場合は、自動保存のあと **Project** タブに自動で戻ります。

### 2.6 タイムラインビュー

Annotation ビュー下部のタイムラインには、記録された全イベントが色分けされた
バーで表示されます。垂直線の再生ヘッドが動画と連動します。

主な操作:

- **ズームイン／ズームアウト** — ズームボタン、またはタイムライン上での
  マウスホイール
- **イベントを選択** — クリックします。選択がハイライトされ、開始時刻と
  終了時刻がステータスバーに表示されます。
- **選択中のイベントを削除** — `Delete` または `Backspace`
- **最後に記録したイベントを取り消す** — `Edit → Undo Last Annotation`
  または `Ctrl + Z`。`RecordingStart` の合成マーカーは取り消し対象から
  除外されます。

タイムラインは再生中に再生ヘッドが画面内に収まるよう自動でスクロールします。

### 2.7 アノテーションを保存する

`File → Export Annotations` を選ぶと、スキーマ v1 の 3 セクションから成る
CSV ファイルを書き出します。

1. **メタデータ** — RABET バージョン、スキーマバージョン、テスト時間
2. **イベントログ** — `Event, Onset, Offset`（秒、小数点以下 4 桁）
3. **行動別サマリー** — `Behavior, Duration, Frequency`

完全な仕様は [`CSV_FORMAT.md`](CSV_FORMAT.md) を参照してください。

保存先に既にファイルがある場合、上書き確認ダイアログが出ます。「いいえ」
を選ぶとダイアログが再表示され、別の名前を選べます。RABET がデータを
無断で消すことはありません。

### 2.8 アノテーションを読み込む

`File → Import Annotations` で過去に書き出した CSV を読み込みます。RABET は
スキーマ v0（1.2.0 より前）と v1 の両方を受け付けます。

`File → Recent Annotations` には直近 10 件のアノテーションファイルが並びます。

イベントが既にロードされている状態で別ファイルを読み込むと、確認ダイアログ
が表示され、確認後に既存のアノテーションがクリアされます。

## 3. 複数の記録ファイルを解析する

**Analysis** ビューは複数のアノテーション CSV をひとつの表に集計し、
スクリプトを書かずに動物・条件・反復セッション間の比較ができます。

```
┌─────────────────────────────────────────────────────────────────┐
│ [Load Files]  [Configure Metrics]   ✓ Enable interval (60 sec) │
├─────────────────────────────────────────────────────────────────┤
│ ┌────────────────┬─────────────────────────────────────────┐    │
│ │   Summary      │  Intervals                              │    │
│ ├────────────────┴─────────────────────────────────────────┤    │
│ │ animal_id │ Attack Latency │ Total Aggression │ ...      │    │
│ │ rat-01    │ 18.43          │ 92.11            │ ...      │    │
│ │ rat-02    │ 25.92          │ 71.30            │ ...      │    │
│ │ ...                                                      │    │
│ └──────────────────────────────────────────────────────────┘    │
│ [Copy to Clipboard]  [Export CSV]  [Visualize Files]            │
└─────────────────────────────────────────────────────────────────┘
```

### 3.1 CSV ファイルを読み込む

2 通りの方法があります。

1. **Load Files** ボタンから 1 つ以上の CSV を選択
2. ウィンドウに CSV をドラッグ&ドロップ

RABET は各ファイルをパースし、ファイル名のステム部分をデフォルトの
`animal_id` として抽出します。続いて、現在のメトリクス設定に従って集計
します。パースに失敗したファイルはログに警告が出力されてスキップされます。

### 3.2 Summary タブ

**Summary** タブには次の列が並びます（1 行 = 1 ファイル）。

- `animal_id`
- 行動ごとに `<行動> Duration` と `<行動> Frequency` の 2 列ずつ
- カスタムメトリクスの列（§3.4）

`Duration` はセッション全体での合計時間（秒）、`Frequency` はイベント回数です。

**Copy to Clipboard** で表全体をクリップボードにコピーします。Excel、JASP、
R、SPSS のいずれにも直接ペーストできる形式です。

### 3.3 Intervals タブ — 時間ビン解析

**Enable interval analysis** にチェックを入れると、各セッションを一定長の
時間ウィンドウ（デフォルト 60 秒）に分割します。Intervals タブは
`(animal, interval)` のペアごとに 1 行を表示します。

解釈の重要ポイント:

- **`Duration`** は当該インターバルと **重なる秒数**。2 つのインターバル
  に跨る行動は、それぞれに按分されます。
- **`Frequency`** は **開始時刻** がインターバル内に入る行動の数。
  複数のインターバルに跨る行動でも 1 回しかカウントされません。

このため、累積率プロットには Duration、イベント数の統計には Frequency が
適しています。

### 3.4 カスタムメトリクスの設定

**Configure Metrics** ボタンでメトリクスエディタを開きます。2 種類の
メトリクスを扱えます。

#### Latency メトリクス（潜時）

セッション開始から、指定した行動の **最初の発生** までの秒数です。
デフォルト設定では:

- *Attack Latency* — 最初の `Attack bites` イベントまでの潜時

必要に応じて好きなだけ追加でき（例: *Sideways threats Latency*）、
個別に有効／無効を切り替えられます。

#### Total-time メトリクス（合計時間）

指定した行動セットの `Duration` の合計です。デフォルト設定では:

- *Total Aggression* — Attack bites + Sideways threats + Tail rattles
  + Chasing
- *Total Aggression (without tail-rattles)* — Attack bites + Sideways
  threats + Chasing

各 Total-time メトリクスに含める行動は編集可能なので、自分の研究に
合った合計（例: *Exploration* = Locomotion + Rearing）を定義できます。

### 3.5 結果をエクスポートする

- **Copy to Clipboard** はアクティブなタブを TSV（タブ区切り）形式で
  クリップボードに書き出します。スプレッドシートに直接ペースト可能です。
- **Export CSV** は Summary 表を `summary_table.csv` として保存します。
  インターバル解析が有効な場合は `summary_intervals.csv` も一緒に書き出します。
- **Visualize Files** は、現在読み込んでいるファイルを引き継いだまま
  Visualization タブへ切り替えます。

## 4. データを可視化する

**Visualization** ビューはラスタープロットを描画します。1 行 = 1 動物
／1 ファイル、1 ティック = 1 イベント、色は行動で分かれます。

### 4.1 ラスタープロットの基本

アノテーション CSV を読み込む（または Analysis から **Visualize Files**
で来る）と、各ファイルが 1 つの横軸トラックになります。イベントは開始
時刻の位置に色付きの縦バーで表示されます。

複数ファイルを初めて読み込んだとき、見やすさのため **デフォルトでは
最初のファイルのみ** が表示されます。残りのファイルは右側のファイル一覧
にチェックが入っていない状態で並びます。

### 4.2 フィルタリング

2 つのチェックボックス群で表示を制御します。

- **ファイル／個体** — 各動物トラックの表示／非表示
- **行動** — 各行動色の表示／非表示。色見本（スウォッチ）が行動ラベルの
  横に表示されます。

サブセット（攻撃行動のみ、条件 A の動物のみ、など）に絞り込む際に、
データを読み直す必要はありません。

### 4.3 色をカスタマイズする

行動名横の色スウォッチをクリックするとカラーピッカーが開きます。
選択した色は即座に適用され、`configs/custom_color_map.json` に保存される
ので、再起動後も維持されます。

デフォルトの配色に戻すには、RABET を閉じてから `custom_color_map.json`
を削除してください。

### 4.4 グリッドと軸範囲

- **垂直グリッド／水平グリッド** — 個別にトグル
- **グリッド色** — グリッド色スウォッチをクリックして変更
- **Max range** — 横軸の最大値（秒）を指定します。5 分セッションと
  10 分セッションを並べて比較するとき、短い方が伸びてしまうのを防げます。

### 4.5 プロットを書き出す

**Export** で現在の表示を **PNG**／**SVG**／**PDF** で保存します。
拡張子から自動判別されます。拡張子を省いた場合は、ダイアログのフィルタ
で選んだ形式が適用されます。

## 5. 評価者間／評価者内の信頼性を評価する

**Reliability** ビューは RABET 自身の出力から一致度メトリクスを計算します。
2 つのモードがあり、ビュー上部のモードバーから切り替えます。

### 5.1 Reliability タブの役割

- **評価者間信頼性** — 2 人の評価者が同じ動画をコーディングし、両者の
  数値がどれくらい一致しているかを知りたい場合に使います。
- **評価者内信頼性** — 1 人の評価者が同じ動画を時間を空けて 2 回コーディング
  し、自分自身の一貫性を確かめたい場合に使います。

同じ Reliability タブで両方を扱います。RABET は 2 つの CSV が同じ評価者
か違う評価者かを区別しません。

### 5.2 Summary モード

**必要な入力。** Analysis ビューから書き出した `summary_table.csv` を
2 ファイル（評価者ごと、またはラウンドごとに 1 つずつ）。各行が 1 個体に
対応し、`animal_id` で識別されます。

**手順。**

1. **Reliability** ビューに切り替えます（`View → Reliability`）。
2. **Summary mode** を選びます。
3. **Load Summary CSV A** で 1 つ目のファイルを読み込みます。
4. **Load Summary CSV B** で 2 つ目のファイルを読み込みます。
5. RABET は `animal_id` で 2 ファイルの動物を突き合わせ、**メトリクス単位**
   で次を計算します。
   - **ICC(2,1)** — Two-way random, 単一測定の級内相関係数
   - **Pearson r** — 線形相関係数
   - **平均絶対差** — 突合した動物に対する `|A − B|` の平均
   メトリクスごとの散布図も自動で描画されます。

**出力。** 結果テーブルは Cicchetti (1994) のバンドで色分けされます。

| ICC | 解釈 | 色 |
| --- | --- | --- |
| ≥ 0.75 | 優秀 | 緑 |
| 0.50 – 0.75 | 中〜良 | 橙 |
| < 0.50 | 不良 | 赤 |

**Export Results** でテーブルを CSV として保存できます。

片方のファイルにしか登場しない動物は、テーブル下の **Unmatched animals**
に列挙されます。意図しないファイル間の不一致を見落とさないための表示です。

### 5.3 Detailed モード

**必要な入力。** **同じ動画** をコーディングした 2 つのアノテーション CSV。
開始時刻と終了時刻が直接比較できる必要があります。

**手順。**

1. Reliability ビューで **Detailed mode** を選びます。
2. アノテーション CSV A と B を読み込みます。
3. **bin width**（時間ビン幅、秒）を設定します。1 秒は瞬間的な行動コーディング
   でよく使われるデフォルト値です。
4. RABET は各アノテーション系列を「時間ビンごとの行動ベクトル」に変換し、
   両ファイルに現れる **すべての行動** について次を計算します。
   - **Cohen's κ** — 偶然の一致を補正した一致度
   - **Krippendorff's α** — 名義尺度として計算したアルファ
   - **生の一致率** — 両評価者が一致するビンの割合
5. ペアワイズ・イベントラスタが行動ごとに 2 段で重ね描きされ、不一致区間
   が一目でわかります。

κ は Landis & Koch (1977) のバンドで色分けされます。

| κ | 解釈 | 色 |
| --- | --- | --- |
| ≥ 0.70 | 高い一致 | 緑 |
| 0.40 – 0.70 | 中程度 | 橙 |
| < 0.40 | 不良 | 赤 |

Krippendorff's α も同じバンドを使います。

### 5.4 解釈の指針

色分けは **スクリーニングの目安** であり、文脈に応じた判断の代替では
ありません。

- Cicchetti の ICC バンド（不良／中／良／優秀の閾値 0.4 / 0.59 / 0.74）は
  臨床測定の文脈で提案されたものです。行動科学の慣例はおおむね近いですが
  分野ごとに揺れがあります。
- Landis & Koch の κ バンドは広く引用されますが **目安** として提案された
  ものに過ぎません。密で変化の早い行動では、優秀な評価者でも 0.4–0.6 に
  落ち着くことがよくあります。

期待より低い一致が出たときは、**Detailed モードのラスタオーバーレイ**
で原因を素早く特定できます。多くは、一方の評価者が常に 1–2 秒遅れて
開始しているといった一定のオフセット、または一方が細分化している
カテゴリの混同です。

### 5.5 R での再現性チェック

`docs/reliability/compute_agreement.R` は Summary モードの統計量
（`ICC(2,1)` は `psych::ICC`、Pearson r は `cor`、平均絶対差）を再現する
独立した R スクリプトです。サニティチェックや、RABET の一致度値を R の
解析パイプラインへ組み込む用途に使えます。

```r
source("docs/reliability/compute_agreement.R")
compute_agreement("rater_a_summary.csv", "rater_b_summary.csv")
```

## 6. プロジェクトモード

RABET の **プロジェクト** は、関連するアセットをひとまとめにする
ディレクトリ + 小さなマニフェストファイルの組み合わせです。

- スコアリング対象の **動画**
- スコアリング結果の **アノテーション CSV**
- スコアリングに用いた **アクションマップ**
- 保存した **解析結果**（サマリーテーブル、インターバルテーブル等）

実験に多数の動画があり、進捗を 1 か所で把握したいときに便利です。

### 6.1 新規プロジェクトの作成

1. `View → Project` で Project モードに切り替えます。
2. **New Project** をクリックします。
3. 親ディレクトリ、プロジェクト名、（任意の）説明文を入力します。

RABET はプロジェクト用サブディレクトリと空のマニフェストファイルを作成
します。Project ビューには（まだ空の）ツリーが表示されます。

### 6.2 ファイルを追加する

**Add Video**、**Add Annotation**、**Add Action Map**、**Add Analysis**
の各ボタンでファイルピッカーが開きます。

- 確認ダイアログで **プロジェクトディレクトリにコピー** するか
  （再現性が高くなり、プロジェクトが自己完結します）、**参照のみ**
  （ファイルは元の場所に残します）を選べます。
- 何度でも追加できます。マニフェストは追加のたびに自動保存されます。

### 6.3 プロジェクト内からアノテーションを付ける

ツリー内の動画を選んで **Annotate** をクリックします。RABET は次を行います。

1. Annotation ビューに切り替え
2. 選んだ動画を読み込み
3. アノテーションをプロジェクトディレクトリ内のファイルに保存

セッションが終わると Project ビューへ自動で戻り、新しいアノテーションが
ツリーに表示されます。

アノテーションのファイル名は人間が読める形式（例:
`rat-01_2026-05-21_001.csv`）になりました。以前のバージョンでは内部の
動画ハッシュが含まれていましたが、1.2.2 で削除されています。

### 6.4 プロジェクトの保存と再オープン

- **保存** は暗黙的に行われます。Add / Remove / Annotate のたびに
  マニフェストが書き直されます。
- **Open Project** でプロジェクトディレクトリを指定します。
- **Close Project** はメモリ上のプロジェクト状態をクリアし、Project
  ビューを開始画面に戻します。

プロジェクトを開いた状態で、そのプロジェクトのアノテーションパスと
衝突するような動画を別途読み込もうとすると、RABET は警告を出して
プロジェクトのアノテーションパスが無断で再利用されることを防ぎます。

## 8. 設定ファイルと永続化される設定

### 8.1 ユーザーデータの保存場所

| OS | パス |
| --- | --- |
| Windows | `%APPDATA%\RABET\` |
| macOS | `~/Library/Application Support/RABET/` |
| Linux | `~/.config/RABET/` |

このルートディレクトリ配下に次のファイルが配置されます。

- `configs/default_action_map.json` — 組み込みデフォルト（初回起動後は
  変更されません）
- `configs/user_action_map.json` — 編集中のアクションマップ。起動時に
  読み込まれるのはこちら
- `configs/default_metrics.json` — 潜時／合計時間メトリクスの設定
- `configs/custom_color_map.json` — 行動別のラスタープロット用色設定
- `logs/rabet_<日付>.log` — 実行ログ

### 8.2 永続化される UI 設定

ウィンドウの位置とサイズ、最後に開いたビュー、テスト時間、ステップ幅、
再生レート、Preserve on rewind フラグ、インターバル解析の設定、最近開いた
ファイル一覧、各ファイルダイアログの直近ディレクトリは、起動間で保持
されます。

### 8.3 デフォルトに戻す

手早くリセットしたい場合:

- **アクションマップ**: `File → Reset Action Map to Default`
- **カスタム色**: `configs/custom_color_map.json` を削除
- **すべて**: RABET を終了し、上記の RABET アプリケーションデータフォルダ
  全体を削除

## 9. CSV ファイル形式

RABET は 3 種類の CSV を書き出します。

- **アノテーション CSV** — Annotation ビューから `File → Export
  Annotations` で出力。メタデータ、イベントログ、行動別サマリーの 3 セクション。
- **サマリー CSV**（`summary_table.csv`）— Analysis ビューから出力。
  1 行 = 1 動物。カスタムメトリクス列を含みます。
- **インターバルサマリー CSV**（`summary_intervals.csv`）— Analysis
  ビューでインターバル解析が有効なときに出力。1 行 = 1 (動物, インターバル) ペア。

3 種類とも `RABET Version` と `Format Schema` のメタデータを含み、
下流ツールがどのバージョンの RABET が書き出したかを判別できます。
時刻は秒単位、小数点以下 4 桁、文字コードは UTF-8 です。

完全な仕様、v0 → v1 互換マトリクス、最小限の pandas パーサーは
[`CSV_FORMAT.md`](CSV_FORMAT.md) を参照してください。

## 10. トラブルシューティング

### 「VLC では再生できるのに、RABET では開けない」

RABET 1.3.2 は次のいずれかを満たすファイルを受け付けます。

1. 拡張子がホワイトリストに入っている
2. ファイルの先頭バイトが既知の動画コンテナと一致する
   （`filetype` ライブラリで判定）
3. PyAV が試験的なオープンに成功する

3 段階すべてに失敗した場合のみエラーを出します。最後の手段として
FFmpeg で変換してみてください。

```bash
ffmpeg -i input.unknown -c copy output.mp4
```

### 「Reliability タブの数値が空白／None になる」

Reliability タブは `pingouin`、`scipy`、`statsmodels`、`krippendorff` に
依存します。配布版バイナリにはこれらが同梱されています。ソースから動かす
場合は次でインストールします。

```bash
pip install pingouin krippendorff
```

`logs/rabet_<日付>.log` にこれらに関する `ImportError` が出ている場合は
配布版から欠落している可能性があるので、ログを添付してイシューを
作成してください。

### 「巻き戻したらイベントが消えた」

デフォルトでは、再生ヘッドが「まだ押されている最中」のイベントの開始
位置を逆方向に超えると、その途中イベントは破棄されます。Recording
controls パネルの **Preserve on rewind** にチェックを入れると保持
されるようになります。

### 「macOS の初回起動時にフォント警告が出る」

macOS では起動時のフォントスキャンで一部のシステムフォントが読み込めない
旨が表示されることがあります。実用上は無害です。matplotlib の他のフォント
データベースは正常に読み込まれ、RABET のウィジェットは Qt のデフォルト
フォントを使います。

### ログファイルの場所

`Log → View Logs` でログディレクトリが OS のファイルマネージャで開きます。
最新のファイルは `rabet_<本日>.log` です。30 日より古いログを一括削除
したい場合は `Log → Clean Up Logs` を使えます。
